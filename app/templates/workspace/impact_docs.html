Impact Compounding Methods

How CISK computes the true importance of each entity in your hierarchy

The Concept

Each entity in the CISK tree (Space, Challenge, Initiative, System, KPI) has an assessed impact level that represents its importance to its parent. The true importance is computed by compounding these levels through the chain from root to leaf.

Space ★★ × Challenge ★★★ × Initiative ★ = True Importance ?

The answer depends on the compounding method chosen. Each method offers a different balance between sensitivity and contrast.

Simple Product Basic

Multiply the numeric weights of each level through the entire chain, then normalize against the maximum possible score at that depth.

score = wspace × wchallenge × winitiative × ...
percentage = score / max_weightdepth
≤ 33% → Level 1  |  ≤ 66% → Level 2  |  > 66% → Level 3
Example: Weights: ★=1, ★★=3, ★★★=5
Space(★★=3) × Challenge(★★★=5) × Initiative(★=1) = 15
Max at depth 3 = 5³ = 125 → 15/125 = 12% → Level 1 (★)

Simple but can be overly punishing — a single weak link drives the result very low due to multiplicative effect.

Geometric Mean Recommended

At each step in the chain, compute the geometric mean of the parent's compounded value and the child's weight. The result naturally stays within the original weight scale.

step1 = wspace
step2 = √(step1 × wchallenge)
step3 = √(step2 × winitiative)
Result mapped to nearest level by weight proximity
Example: Weights: ★=1, ★★=3, ★★★=9 (Toyota scale)
step1 = 3 (Space ★★)
step2 = √(3 × 9) = √27 = 5.2 → nearest to ★★ (w=3) or ★★★ (w=9) → ★★
step3 = √(5.2 × 1) = 2.3 → nearest to ★★ (w=3) → ★★

Balanced — a weak link reduces importance but doesn't obliterate it. Based on the CISK simplified compounding algorithm.

Toyota QFD Strongest Contrast

Uses a fuzzy lookup table inspired by Toyota's Quality Function Deployment (QFD). At each step, the parent and child levels are looked up in a predefined matrix. This method produces the sharpest differentiation — a weak link in the chain has a dramatic effect.

Parent \ Child ★★★ ★★ ★ Not set
★★★ ★★★ ★★★ ★★ —
★★ ★★★ ★★ ★ —
★ ★★ ★ — —
Not set — — — —
Example: Space(★★) × Challenge(★★★) → lookup(★★, ★★★) = ★★★
Then ★★★ × Initiative(★) → lookup(★★★, ★) = ★★
Final true importance: ★★

No numeric weights needed. Maximum contrast enforces clear prioritization — avoids "everything is medium" syndrome. Ideal for executive decision-making.

Customizable: The lookup matrix can be edited in the Impact Scale admin page. You can adjust how parent and child levels combine to match your organization's decision-making style.

Toyota Weighted (downstream)

Uses exponential reinforcement on the child (deepest entity) rating. Higher ratings become disproportionately more important:

LevelReinforced Weight
★1
★★2
★★★4

Parent weights use the configured scale as-is. The product is normalized against the maximum possible score and mapped to ★/★★/★★★.

Example: Space(★★, w=2) × Challenge(★★★, w=3) × Initiative(★★★, reinforced=4)
Score = 2 × 3 × 4 = 24, Max = 3 × 3 × 4 = 36, Ratio = 67% → ★★★

Toyota Weighted (full) Max Amplification

Applies exponential reinforcement to ALL levels in the chain — both upstream (parents) and downstream (child). This creates maximum differentiation between high and low importance chains.

Example: Space(★★★, r=4) × Challenge(★★, r=2) × Initiative(★★★, r=4)
Score = 4 × 2 × 4 = 32, Max = 4 × 4 × 4 = 64, Ratio = 50% → ★★

Compare with: Space(★★★, r=4) × Challenge(★★★, r=4) × Initiative(★★★, r=4)
Score = 4 × 4 × 4 = 64, Max = 64, Ratio = 100% → ★★★

Inspired by Toyota's weighting model where high-priority items receive exponentially more attention. One weak link in the chain significantly reduces the final score.

Configurable: The reinforcement weights (default: ★=1, ★★=2, ★★★=4) can be edited in the Impact Scale admin page. Increase the ratio for sharper differentiation (e.g. 1, 3, 9).

Simple Product and Toyota Weighted methods push low chains down aggressively. Geometric Mean and Toyota QFD preserve more nuance. Toyota Weighted (full) provides the strongest differentiation — only consistently high chains get ★★★.

⚠ No Consensus Special State

When assessors cannot agree on an entity's impact level, the "No consensus" option can be selected instead of a star rating. This requires a disagreement note documenting who disagreed and why.

Effect on True Importance: No consensus is treated as "not set" — it breaks the compounding chain. The entity and all its descendants will show no True Importance value (—), regardless of which calculation method is configured.

This ensures that no misleading rollup numbers are produced when there is genuine disagreement about an entity's importance. The warning badge (⚠) appears in the workspace tree to make unresolved disagreements visible. Clicking the badge shows the disagreement note.

Workspace Tree Display

  • Inline badge: An outlined ⚠ badge appears next to the entity name
  • Impact column: A filled ⚠ badge replaces the star rating
  • Popup: Click either badge to see the disagreement note and change the assessment

Configuration

The color of the No Consensus badge can be customized in Workspace Administration → Impact Scale under "Special State Colors". The default color is amber (#f59e0b).
